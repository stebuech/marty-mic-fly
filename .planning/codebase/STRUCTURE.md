## Naming Conventions

**Files:**
- snake_case.py for all Python modules
- UPPERCASE.md for important project files (CLAUDE.md)
- lowercase_with_underscores.yaml for configs

**Directories:**
- snake_case for all directories
- Plural for collections: `notes_and_helpers/`, `results/`

**Special Patterns:**
- `*_comparison.py` - Comparative analysis scripts
- `*_analysis.py` - End-to-end pipelines
- `*_calculation.py` - Specific computation utilities
- `*_validation.py` - Metrics and performance evaluation

## Where to Add New Code

**New Data Source:**
- Primary code: `daq.py` (add class alongside ESCTelemtry, MicArray)
- Integration: Update `DAQ.__init__()` and `_start_recording()`
- Config: Add CLI arguments in `main()`

**New Analysis Module:**
- Implementation: `analysis/{name}.py`
- Config: `analysis/{name}_config.yaml` if complex
- Entry: Add argparse CLI in the module

**New Hardware Control:**
- Implementation: New file in root (e.g., `servo_control.py`)
- Note: May need separate process due to CPU affinity

**Utilities:**
- Shared helpers: Create `utils/` directory (doesn't exist yet)
- One-off scripts: `notes_and_helpers/`
